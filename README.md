# The Evergreen Rules (Core Programming Standards)

A curated set of AI agent instructions focused on **timeless programming principles** rather than transient framework syntax. 

While tools like `cursor.directory` provide excellent category-specific snippets, this repository focuses on the "proven strategies" that have survived decades of industry evolution, inspired by the work of **Robert C. Martin (Uncle Bob)**, **Martin Fowler**, and **Douglas Crockford**.

## 🎯 The Philosophy
Most AI-generated code is "correct" but "messy." It works, but it’s hard to change. These rules force the AI to act as a **Senior Architect** who prioritizes maintainability, readability, and reliability.

### Key Pillars:
- **Clean Code (Martin):** Prioritize human readability and small, focused units.
- **Refactoring (Fowler):** Continuous improvement through small, behavior-preserving steps.
- **The Good Parts (Crockford):** Reliability through minimalism and avoiding language "traps."
